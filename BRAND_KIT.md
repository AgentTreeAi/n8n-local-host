## 1 — The 5,000 ft view

**AgentTree builds custom AI systems and workflow automation for small and medium businesses.**
We take the manual, repetitive, revenue-leaking work a business runs on — chasing leads, cleaning
the CRM, following up, scheduling — and turn it into systems that run themselves.

The promise in one line:

> ### "Turn your business into a self-running system."

We don't sell software seats or generic templates. We **architect and deploy bespoke AI agents and
automation pipelines directly into the tools a business already uses**, then operate them like
infrastructure: monitored, tuned, and expanded as the business grows.

### The core thesis

> ### "The problem isn't effort. It's infrastructure."

Most growing businesses aren't failing because the team isn't working hard enough. They're failing
because leads fall through the cracks, tools don't talk to each other, and follow-up depends on
someone remembering. AgentTree fixes the infrastructure so growth stops breaking the business.

## 6 — How we work (engagement model)

The canonical three-beat engagement narrative is **Map → Build → Operate.** Use these three nouns
consistently across all surfaces.

| Phase | When | What happens |
|---|---|---|
| **01 · Map** | Week 0 | Free 30-min working session. Surface the highest-leverage automation moves and outline a system blueprint. |
| **02 · Build** | Weeks 1–4 | Design, integrate, and deploy the custom AI system directly into the existing stack. No rip-and-replace. |
| **03 · Operate** | Ongoing | Treat the system like infrastructure — continuous tuning, monitoring, and expansion as the business scales. |

### Engineering-grade deployment detail (inside "Build")

For surfaces that need more granularity, Build decomposes into an engineering pipeline:

1. **System Audit & Diagnosis** — map workflows, find revenue leaks, calculate ROI.
2. **Architecture Design** — define triggers, select models, design safeguards before any code.
3. **Integration & Deployment** — build into existing infrastructure with shadow-testing for zero disruption.
4. **Continuous Optimization** — monitor performance, tune prompts, expand capabilities.

> **Consistency rule:** Public-facing process = **3 beats (Map / Build / Operate).** The 4-step
> pipeline lives *inside* Build for teams that want the engineering detail. Don't present two
> competing step counts on the same journey.

### Implementing on-brand

- Colors are CSS custom properties consumed by Tailwind. Use semantic utilities
  (`bg-background`, `text-foreground`, `border-border`) for theme-aware surfaces and `brand-*`
  utilities (`text-brand-sage`, `bg-brand-terracotta`) for explicit brand accents.
- Default theme is **dark** (`defaultTheme="dark"`, system theme disabled).
- Use the typography utilities (`text-display-1`, `text-lead`, `text-eyebrow`) rather than ad-hoc
  font sizing on marketing surfaces.
- Pull logos through `lib/brand-assets.ts` (`primaryLogo`, `logos.*`) and the `cld()` transform
  helper — never hardcode Cloudinary URLs.
- Always wire `prefers-reduced-motion` for any new animation (the global reset already handles
  declarative CSS animations).
